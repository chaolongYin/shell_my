背景

目前解析dump的方式有较多钟，比如:

1:MTK minidump(kernel elf coredump)<<<<<<配合arm(64) gdb来使用，有兴趣的，可以找找我历史的wiki

2:QCOM fulldump (DDR img) <<<<<< 配合linux-ramdump-parser-v2,然后再配合T32 simulator使用

今天再介绍一个出自redhat的解析工具，可以对minidump,fulldump,以及内核标准的kdump img进行解析

redhat基本知识准备

redhat　crash原理

这里需要知道两大鼎鼎大名的工具:qemu+gdb，gdb不多说了，有兴趣的，可以找找历史wiki，或者度娘;

qemu是一个开源的多架构模拟器，市面上常见的cpu架构都支持，主要用途：开源虚拟机，交叉调试等，

比如android发布时，使用的模拟器就是基于qemu的，那么这里redhat的crash工具，就是基于qemu+gdb

的基础上添加各种debug调试而来

redhat crash下载编译

两个比较重要的网址：

a)代码下载

1:http://people.redhat.com/anderson/help.html <<<< src code下载/帮助，官网网页

2:https://github.com/crash-utility/crash/releases　<<<<<src　code下载，

b)编译

我们现在主要做arm64的产品，所有我们编译用于arm64 debug的版本 


make target=ARM64

make extensions target=ARM64

sudo make install

正常情况下，编译完成后，就可以使用crash了


crash example

这里以qcom arm64一个fulldump为例子

文件准备：死机后fulldump + vmlinux

启动crash: crash ../vmlinux DDRCS0.BIN@0x20000000,DDRCS1.BIN@0x80000000

其中DDRCS0.BIN, DDRCS1.BIN分别是两片ddr的完整镜像拷贝，根据产品配置不一样

可能只有一个文件，也可能有多个文件,其后面@跟上的地址,源自内存的地址分布


使用命令log查看kernel log



使用命令ps查看kernel log


使用命令struct查看具体结构体的值


crash封装的命令还有很多:比如查看timer,查看irq，查看list等


前面有提到crash,核心引擎使用gdb，基本上所有的gdb命令在crash中也是可以使用的

比如gdb set substitute-path path1 path2，用于服务器包代码路径映射到本地路径等
